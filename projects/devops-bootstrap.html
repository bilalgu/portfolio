DevOps Bootstrap Stack – Version 1.5.0

Bilal – Systems & Cybersecurity Engineer – 23 years old – Master’s degree (Work-Study) GitHub: https://github.com/bilalgu/devops-bootstrap

This personal DevOps/Cyber project aims to build a secure, scalable and automated cloud infrastructure — as close as possible to real production constraints.

It serves as :

  • a public technical portfolio
  • a learning environment for DevOps & Cloud engineering
  • a reusable stack to bootstrap real-world deployments

Version 1.5.0 – Multi-Service Stack with Reverse Proxy

This release includes the deployment of a complete Web/API/DB stack, routed via Traefik reverse proxy, all automated from provisioning to deployment.

Core Features

Layer Technology
IaaS AWS EC2 (Ubuntu 22.04)
IaC Terraform
Configuration Ansible
Containerization Docker + Docker Compose
Reverse Proxy Traefik
Services Web (Nginx), API (Node.js), DB (PostgreSQL)
Security SSH hardening, nftables firewall, fail2ban
CI/CD GitHub Actions

What’s Deployed

Step 1 – Infrastructure Provisioning

  • EC2 instance provisioned via Terraform
  • SSH key and security group (ports 22 and 80 only)

Step 2 – System Configuration with Ansible

  • Docker & Docker Compose installed
  • nftables firewall injected at boot using systemd
  • Fail2ban configured with EC2-specific logs

Step 3 – Multi-Service Stack Deployment

  • Docker Compose stack:
    • web: static frontend (Nginx)
    • back: Node.js API
    • db: PostgreSQL with persistent volume
  • Reverse proxy traefik handles all HTTP traffic

Step 4 - Continuous Deployment (GitHub Actions)

  • Redeploys automatically (if the VM is up)
  • GitHub Actions triggers on changes to Ansible or Docker files
  • Automatic deployment to the live EC2 server (via SSH)

What I’ve Learned
