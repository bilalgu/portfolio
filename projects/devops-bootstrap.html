This DevOps & Cloud Security project aims to build a realistic, secure and automated cloud infrastructure — as close as possible to real production constraints.

It serves as:

  • a public technical portfolio (DevOps/Cyber/Cloud)

  • a learning platform for professional-grade practices

  • a reusable stack to bootstrap real-world deployments

Version 2.0.0 – Multi-Service Stack with Security & Monitoring

This release completes the roadmap for version 2.0.0, introducing:

  • a realistic Web/API/DB architecture

  • a secure and hardened EC2 instance

  • live metrics via cAdvisor & Prometheus

  • full automation with Terraform + Ansible

Core Technologies

Layer Tech Stack
IaaS AWS EC2 (Ubuntu 22.04)
IaC Terraform
Provisioning Ansible
Containerization Docker + Docker Compose
Routing Traefik (HTTP reverse proxy)
Services Web (Nginx), API (Node.js), DB (PostgreSQL)
Security SSH hardening, fail2ban, nftables firewall
Monitoring cAdvisor, Prometheus (routed via Traefik)
CI/CD GitHub Actions

What’s Deployed

Infrastructure

  • EC2 instance provisioned with SSH key and firewall rules

  • nftables firewall injected via systemd (persistent across reboots)

Application Stack

  • Docker Compose stack (web, back, db) deployed with healthchecks

  • Traefik reverse proxy routes:

    • / → Web frontend

    • /api/* → Node.js backend

    • /monitoring/* → Prometheus

Monitoring

  • cAdvisor exposes container and host metrics

  • Prometheus scrapes metrics from cAdvisor (5s interval)

  • Prometheus is available via /monitoring through Traefik

Automation & CI

  • Everything deploys via Ansible in one command

  • GitHub Actions CI redeploys automatically on changes
